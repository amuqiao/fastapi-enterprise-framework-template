## 12.6 术语表

### 12.6.1 核心术语

| 术语 | 英文 | 描述 |
|------|------|------|
| 镜像 | Image | Docker 镜像，是一个只读的模板，用于创建容器 |
| 容器 | Container | Docker 容器，是镜像的运行实例 |
| 仓库 | Registry | 存储和分发 Docker 镜像的服务 |
| 卷 | Volume | 用于持久化存储容器数据的机制 |
| 网络 | Network | 用于容器通信的网络配置 |
| Dockerfile | Dockerfile | 用于构建 Docker 镜像的文本文件 |
| docker-compose | Docker Compose | 用于定义和运行多容器 Docker 应用的工具 |
| 多阶段构建 | Multi-stage Build | 从单个 Dockerfile 构建多个阶段，减小最终镜像体积 |
| 命名空间 | Namespace | 用于隔离容器资源的 Linux 内核特性 |
| 控制组 | Cgroups | 用于限制容器资源使用的 Linux 内核特性 |
| 存储驱动 | Storage Driver | 用于管理容器镜像和容器层的驱动 |

### 12.6.2 高级术语

| 术语 | 英文 | 描述 |
|------|------|------|
| 服务 | Service | 在 Docker Compose 中定义的一个应用组件 |
| 栈 | Stack | 在 Docker Compose 中定义的一组服务 |
| Swarm | Swarm | Docker 内置的容器编排工具 |
| Kubernetes | Kubernetes | 生产级容器编排平台 |
| 服务发现 | Service Discovery | 自动发现和注册服务的机制 |
| 负载均衡 | Load Balancing | 在多个容器之间分配流量的机制 |
| 健康检查 | Health Check | 用于检查容器是否正常运行的机制 |
| 滚动更新 | Rolling Update | 逐步更新容器的机制，确保服务不中断 |
| 蓝绿部署 | Blue-Green Deployment | 同时运行两个版本的应用，切换流量实现零停机部署 |
| 金丝雀发布 | Canary Release | 将新版本逐步部署到生产环境，验证后再全面发布 |
