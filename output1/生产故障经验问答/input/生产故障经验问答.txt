# 生产故障经验问答

# 1\.同个服务节点反复出问题（cpu\\内存高）？

- 可能是job引起，因为只有job会把流量指向同一个节点执行。

# 2\.如何知道这个节点可能出问题的原因？

- 可以通过查询这个节点的日志，看正在做这么事情，从中找到一些思路：

```Bash
#通过LTS查询指定节点日志
appName : jgcloud-dos AND podName : jgcloud-dos-5cdf88c8b-dlq2x
```

- 可以通过数据库监控，看是否加载了很多数据。

# 3\.日志看到sql正常执行了，但是数据最终没有被修改，又被还原了

可能是事务回滚导致的，需要结合日志和代码排查验证。
